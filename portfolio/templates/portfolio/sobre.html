1. Arquitetura MVT do Django

A arquitetura MVT (Model-View-Template) é o padrão de desenho de software que o Django utiliza para separar as responsabilidades de dados, lógica e apresentação.

Desenho Manual da Arquitetura

Representação do ciclo de vida de um pedido HTTP no Django.

Model

Representa a estrutura dos dados da aplicação. É a única fonte de informação sobre os teus dados e contém os campos e comportamentos essenciais do que está armazenado na Base de Dados.

View

É a "ponte lógica". Recebe pedidos HTTP do utilizador, processa a lógica necessária (consultar a base de dados, gerir formulários) e decide que resposta ou Template apresentar de volta.

Template

É a camada de apresentação. Trata-se essencialmente de HTML misturado com a linguagem de template do Django, gerando dinamicamente o que o utilizador final visualiza no navegador.

Fluxo de Informação

O fluxo começa quando o Utilizador faz um pedido no Browser. O Django consulta o URL para direcionar o pedido para a View correta. A View comunica com o Model para buscar/gravar dados, envia-os para o Template preencher o HTML dinâmico e, finalmente, devolve a resposta HTTP ao Browser do utilizador.

3. API RESTful com Django Ninja

Esta aplicação disponibiliza uma API RESTful dedicada à gestão de receitas culinárias. Uma API RESTful expõe recursos através de endereços HTTP e utiliza métodos como GET, POST, PUT e DELETE para consultar, criar, atualizar e apagar dados.

A API foi construída com Django Ninja, uma biblioteca que permite definir endpoints tipados em Python e gerar automaticamente documentação interativa no formato OpenAPI. Os recursos disponíveis são receitas, ingredientes e utilizadores.

9. Arquitetura da Solução em Produção e Pipeline CI/CD

Para além do ciclo de funcionamento local, a aplicação foi arquitetada para correr de forma profissional e resiliente na nuvem de produção. Mapeámos e desenhámos à mão o fluxo que liga o computador local ao servidor final.

Esquema Arquitetural e Pipeline CI/CD (Desenho à Mão no Papel)

Diagrama que ilustra a integração do desenvolvimento local, controlo com Git/GitHub, empacotamento com Docker, armazenamento cloud de dados e media, e o deploy automatizado.

Detalhamento dos 7 Componentes Principais:

1. PC Local (VS Code & Git)

O ambiente local onde o código é produzido. O Git regista cada alteração em commits e ramificações de segurança.

2. Servidor Django Local

O ambiente de execução local (usando SQLite) onde testamos e validamos as páginas dinâmicas em http://127.0.0.1:8000.

3. Repositório GitHub

Atua como o "Hub de sincronização". O código é empurrado via git push para o repositório remoto.

4. Pipeline CI/CD

Processo de automatização. Um robô em nuvem monitoriza os pushes, executa testes e despoleta o deploy seguro no servidor final.

5. Contentor Docker

Garante a portabilidade absoluta do sistema. A aplicação e dependências (pip, django, cloudinary, etc.) correm de forma isolada dentro de um contentor idêntico em qualquer máquina.

6. Servidores Cloud de Dados e Media

Neon Postgres: Base de dados SQL escalável na cloud que gere toda a persistência de dados.
Cloudinary: Armazenamento cloud externo seguro onde guardamos as imagens e ficheiros estáticos (com WhiteNoise).

7. Browser do Cliente

O utilizador acede ao site através do link público. O tráfego HTTP é processado e a aplicação dinâmica é desenhada no ecrã de forma responsiva.
