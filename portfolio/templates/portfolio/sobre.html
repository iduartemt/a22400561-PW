1. Arquitetura MVT do Django

A arquitetura MVT (Model-View-Template) é o padrão de desenho de software que o Django utiliza para separar as responsabilidades de dados, lógica e apresentação.

Desenho Manual da Arquitetura

Representação do ciclo de vida de um pedido HTTP no Django.

Model

Representa a estrutura dos dados da aplicação. É a única fonte de informação sobre os teus dados e contém os campos e comportamentos essenciais do que está armazenado na Base de Dados.

View

É a "ponte lógica". Recebe pedidos HTTP do utilizador, processa a lógica necessária (consultar a base de dados, gerir formulários) e decide que resposta ou Template apresentar de volta.

Template

É a camada de apresentação. Trata-se essencialmente de HTML misturado com a linguagem de template do Django, gerando dinamicamente o que o utilizador final visualiza no navegador.

Fluxo de Informação

O fluxo começa quando o Utilizador faz um pedido no Browser. O Django consulta o URL para direcionar o pedido para a View correta. A View comunica com o Model para buscar/gravar dados, envia-os para o Template preencher o HTML dinâmico e, finalmente, devolve a resposta HTTP ao Browser do utilizador.
